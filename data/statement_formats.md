## Part 1: CSVフォーマット定義

### 銀行CSV

#### 1. みずほ銀行
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: なし（または「お取引日,摘要,お引き出し金額,お預け入れ金額,残高」）
列順: 日付, 摘要, 出金, 入金, 残高
日付形式: YYYY/MM/DD
金額: カンマなし整数
取引先: 摘要列（カナ混在）
特記: BOM なし、ネットバンキング版は4列構成もあり
```

#### 2. 三菱UFJ銀行
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: あり（「取引日,摘要,支払い金額,預かり金額,差引残高」）
列順: 取引日, 摘要, 支払い金額, 預かり金額, 差引残高
日付形式: YYYY/MM/DD
金額: カンマあり（例: 1,234,567）
取引先: 摘要列（全角カナ混在）
特記: 法人口座は「摘要2」列が追加されることがある
```

#### 3. 三井住友銀行
```
文字コード: UTF-8（BOMなし）
改行: LF
ヘッダー行: あり（「取引年月日,摘要,支払金額,預り金額,残高」）
列順: 取引年月日, 摘要, 支払金額, 預り金額, 残高
日付形式: YYYY/MM/DD
金額: カンマなし
取引先: 摘要列
特記: 個人口座と法人口座で若干列名が異なる場合あり
```

#### 4. りそな銀行
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: あり（「年月日,摘要,出金金額,入金金額,残高」）
列順: 年月日, 摘要, 出金金額, 入金金額, 残高
日付形式: YYYY/MM/DD
金額: カンマなし
取引先: 摘要列（半角カナ）
特記: 埼玉りそな・関西みらい銀行も同形式
```

#### 5. 楽天銀行
```
文字コード: UTF-8（BOM付き）
改行: CRLF
ヘッダー行: あり（「取引日,入出金区分,取引内容,出金,入金,残高,メモ」）
列順: 取引日, 入出金区分, 取引内容, 出金, 入金, 残高, メモ
日付形式: YYYY/MM/DD
金額: カンマなし
取引先: 取引内容列（ハイフン区切りの複合情報）
特記: 入出金区分列で「入金」「出金」「振込」等を明示
```

#### 6. PayPay銀行
```
文字コード: UTF-8（BOMなし）
改行: LF
ヘッダー行: あり（「日付,摘要,受取金額,支払金額,残高」）
列順: 日付, 摘要, 受取金額, 支払金額, 残高
日付形式: YYYY/MM/DD
金額: カンマなし
取引先: 摘要列（英数・カナ混在）
特記: 旧ジャパンネット銀行時代の形式と同一
```

#### 7. ゆうちょ銀行
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: あり（「年月日,お取扱内容,お支払金額,お預り金額,差引残高」）
列順: 年月日, お取扱内容, お支払金額, お預り金額, 差引残高
日付形式: YYYY/MM/DD
金額: カンマなし
取引先: お取扱内容列（全角）
特記: 期間指定でCSVダウンロード。通常100件上限あり
```

#### 8. 住信SBIネット銀行
```
文字コード: UTF-8（BOMなし）
改行: LF
ヘッダー行: あり（「取引日,内容,出金金額,入金金額,残高」）
列順: 取引日, 内容, 出金金額, 入金金額, 残高
日付形式: YYYY/MM/DD
金額: カンマなし
取引先: 内容列
特記: ハイブリッド預金は別口座扱い
```

### カードCSV

#### 1. 三菱UFJカード
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: あり
列順: 利用日, 利用店名, 利用者, 支払方法, 利用金額, 今回払い金額
日付形式: YYYY/MM/DD
金額: カンマあり
特記: 分割払いは「支払方法」列に「分割○回」と表記
```

#### 2. 三井住友カード
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: あり
列順: 利用日, 利用店名・商品名, お支払い金額, 利用者名, 支払区分, 今月の分割支払金額等
日付形式: YYYY/MM/DD
金額: カンマあり
特記: Vポイント・キャッシュバック行は金額が負数
```

#### 3. JCBカード
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: あり
列順: 年月日, 利用店名, 本人金額, 家族金額, 分類コード
日付形式: YYYY/MM/DD
金額: カンマなし
特記: 本人・家族で列が分かれる。ポイント充当は別行
```

#### 4. 楽天カード
```
文字コード: UTF-8（BOM付き）
改行: CRLF
ヘッダー行: あり
列順: 利用日, 利用店名・商品名, 支払方法, 利用金額, 当月支払い金額, ポイント
日付形式: YYYY/MM/DD
金額: カンマなし
特記: 楽天ポイント利用は「利用金額」がマイナスで表示されることがある
```

#### 5. アメリカン・エキスプレス
```
文字コード: UTF-8（BOM付き）
改行: CRLF
ヘッダー行: あり
列順: 日付, 説明, カード, 金額
日付形式: YYYY/MM/DD
金額: カンマなし（クレジット分は先頭に「-」）
特記: 英語・日本語混在。外貨建て取引は元通貨・換算後を別列で表示
```

#### 6. セゾンカード
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: あり
列順: 利用年月日, 利用店名, 支払区分, 利用額, 支払額
日付形式: YYYY/MM/DD
金額: カンマなし
特記: 永久不滅ポイントの充当は別行記載。セゾンアメックスも同形式
```

#### 7. dカード
```
文字コード: UTF-8（BOM付き）
改行: CRLF
ヘッダー行: あり
列順: 利用年月日, 利用店名・商品名, 利用金額, 今回のお支払い金額, 支払方法
日付形式: YYYY/MM/DD
金額: カンマなし
特記: dポイントを使った場合は「利用金額」と「お支払い金額」が乖離する
```

#### 8. イオンカード
```
文字コード: Shift_JIS
改行: CRLF
ヘッダー行: あり
列順: 利用日, ご利用先名称, 支払区分, ご利用金額, 今回お支払い金額
日付形式: YYYY/MM/DD
金額: カンマなし
特記: WAONポイント充当分は負数行として追記
```

## Part 2: PDFフォーマット定義

### 銀行 PDF明細書

#### 1. みずほ銀行
```
向き: 縦（A4ポートレート）
構成: 口座情報ヘッダー（口座番号・期間）→ 明細テーブル → フッター
列順: お取引日 / 摘要 / お引き出し金額 / お預け入れ金額 / 残高
日付表記: YYYY年MM月DD日
金額表記: カンマあり（例: 1,234,567円）
マイナス表記: マイナスなし（出金列・入金列を分ける構成）
取引先位置: 摘要列（1行またはカナ2行構成）
口座番号位置: PDFヘッダー左上・右上に支店名+口座番号
ページフッター: ページ数（X/Y）、発行日時
DL形式の特徴: みずほダイレクト → ページ単位PDF。帳票はA4縦、行数上限あり（30行/ページ）
```

#### 2. 三菱UFJ銀行
```
向き: 縦（A4ポートレート）
構成: ヘッダー（銀行ロゴ左上、口座名義・番号右）→ 明細テーブル → 合計行 → フッター
列順: 取引日 / 摘要 / 支払い金額 / 預かり金額 / 差引残高
日付表記: YYYY/MM/DD
金額表記: カンマあり（例: 1,234,567）
マイナス表記: マイナスなし（支払/預かりを分ける）
取引先位置: 摘要列（全角カナ表記多い）
口座番号位置: ヘッダー右側（支店名・口座種別・口座番号を段組み）
ページフッター: 「三菱UFJ銀行」ロゴ + ページ番号
DL形式の特徴: 三菱UFJダイレクト → 「残高照会・入出金明細照会」から期間指定PDF
```

#### 3. 三井住友銀行
```
向き: 縦（A4ポートレート）
構成: ロゴ（SMBCブルー）→ 口座情報 → 明細テーブル → ページフッター
列順: 取引年月日 / 摘要 / 支払金額 / 預り金額 / 残高
日付表記: YYYY/MM/DD
金額表記: カンマなし
マイナス表記: なし（支払/預りで区別）
取引先位置: 摘要列（2行折り返しあり）
口座番号位置: ヘッダー中央上部（「普通 XXXXXXX」形式）
ページフッター: 三井住友銀行ロゴ + 問い合わせ番号
DL形式の特徴: Olive / SMBCダイレクトから帳票PDF。1PDFあたり最大500件
```

#### 4. りそな銀行
```
向き: 縦（A4ポートレート）
構成: ヘッダー（りそなグループロゴ右上）→ 期間・口座情報 → 明細テーブル → フッター
列順: 年月日 / お取り扱い内容 / お支払い金額 / お預かり金額 / 残高
日付表記: YYYY/MM/DD
金額表記: カンマあり
マイナス表記: なし
取引先位置: お取り扱い内容列（半角カナ）
口座番号位置: ヘッダー左（「普通預金 XXXXXXX」）
ページフッター: ページ番号 + 支店電話番号
DL形式の特徴: りそなビジネスダイレクト → 期間別PDF。最大1,000件表示
```

#### 5. 楽天銀行
```
向き: 縦（A4ポートレート）
構成: 楽天ロゴ（赤）→ 口座情報（氏名・口座番号）→ 明細テーブル
列順: 取引日 / 取引内容 / 出金金額 / 入金金額 / 残高
日付表記: YYYY/MM/DD
金額表記: カンマあり
マイナス表記: なし（出金列で表現）
取引先位置: 取引内容列（英数・カナ混在）
口座番号位置: ヘッダー右（普通預金 XXXXXXX）
ページフッター: 楽天銀行株式会社 + 免責事項
DL形式の特徴: 楽天銀行アプリ or PC → 入出金明細のPDF出力。1ページに最大50件
```

#### 6. PayPay銀行
```
向き: 縦（A4ポートレート）
構成: PayPayロゴ（黄黒）→ 口座情報 → 明細テーブル → フッター
列順: 取引日 / 取引内容 / 受取 / 支払 / 残高
日付表記: YYYY/MM/DD
金額表記: カンマなし
マイナス表記: なし
取引先位置: 取引内容列
口座番号位置: ヘッダー左（「普通 XXXXXXX」）
ページフッター: PayPay銀行株式会社 + ページ番号
DL形式の特徴: PayPayアプリ or PC → 月次PDF。シンプルなモノクロ帳票
```

#### 7. ゆうちょ銀行
```
向き: 縦（A4ポートレート）
構成: ゆうちょロゴ（緑）→ 記号番号（口座番号）→ 明細テーブル → フッター
列順: 年月日 / お取扱内容 / お支払金額 / お預り金額 / 差引残高
日付表記: YYYY年MM月DD日
金額表記: カンマあり
マイナス表記: なし
取引先位置: お取扱内容列（全角）
口座番号位置: ヘッダー上部（「記号 XXXXX 番号 XXXXXXXX」形式）
ページフッター: ゆうちょ銀行 + 日本郵便株式会社 クレジット表記
DL形式の特徴: ゆうちょダイレクト → 入出金明細照会から期間指定PDF。旧来型の縦帳票デザイン
```

#### 8. 住信SBIネット銀行
```
向き: 縦（A4ポートレート）
構成: SBIロゴ（青）→ 口座情報 → 明細テーブル
列順: 取引日 / 取引内容 / 出金金額 / 入金金額 / 残高
日付表記: YYYY/MM/DD
金額表記: カンマなし
マイナス表記: なし
取引先位置: 取引内容列（英数混在）
口座番号位置: ヘッダー左上
ページフッター: 住信SBIネット銀行株式会社 + ページ番号
DL形式の特徴: Web / アプリから月次PDF出力。ハイブリッド預金等は口座別に別PDF
```

### カード PDF明細書

#### 1. 三菱UFJカード
```
向き: 縦（A4ポートレート）
構成: カードロゴ → 請求月・請求金額サマリー → 明細テーブル → 小計 → フッター
列順: 利用日 / 利用店名・商品名 / 利用者 / 支払方法 / 利用金額 / 当月支払額
利用店舗位置: 利用店名列（2行折り返しあり）
日付表記: YYYY/MM/DD
金額表記: カンマあり
分割払い表記: 「支払方法」列に「2回払い」「3回均等分割」等
リボ表記: 「リボルビング払い（残高XXXXX円）」と明記
ポイント行: 「三菱UFJカードポイント還元」として負数行で記載
キャッシュバック行: なし（ポイント系のみ）
```

#### 2. 三井住友カード
```
向き: 縦（A4ポートレート）
構成: SMBCグループロゴ → 請求月・お支払い合計 → 明細テーブル → フッター
列順: ご利用日 / ご利用店名・商品名 / お支払金額 / 今回のお支払い金額 / ご利用者名 / お支払い方法
利用店舗位置: ご利用店名列（日本語・英語混在）
日付表記: YYYY/MM/DD
金額表記: カンマあり
分割払い表記: 「お支払い方法」列に「分割X回」
リボ表記: 「マイ・ペイすリボ」として記載。元本・手数料を分けた行で表示
ポイント行: Vポイント還元は「Vポイントサービス」として負数行
キャッシュバック行: キャッシュバックサービス利用時は「キャッシュバック」行で負数
```

#### 3. JCBカード
```
向き: 縦（A4ポートレート）
構成: JCBロゴ（青・赤）→ ご請求明細 → 合計欄 → フッター
列順: ご利用年月日 / ご利用店名 / お支払区分 / お支払い回数 / ご利用金額 / 割引・手数料
利用店舗位置: ご利用店名列
日付表記: YYYY年MM月DD日
金額表記: カンマあり（例: ¥12,345）
分割払い表記: 「お支払区分」に「分割」、「お支払い回数」に回数
リボ表記: 「リボ払い」として区分列に記載。手数料は別行
ポイント行: OkiDokiポイント充当は「OkiDoki充当」として負数行
キャッシュバック行: なし
```

#### 4. 楽天カード
```
向き: 縦（A4ポートレート）
構成: 楽天カードロゴ → ご請求金額サマリー → 明細テーブル → 楽天ポイント明細 → フッター
列順: 利用日 / 利用店名・商品名 / 支払方法 / 利用金額 / 当月支払い金額
利用店舗位置: 利用店名列（英数混在、楽天市場系は長い）
日付表記: YYYY/MM/DD
金額表記: カンマなし
分割払い表記: 「支払方法」に「分割X回払い」
リボ表記: 「楽Pay（リボ）」として記載。月額固定額とは別に手数料行あり
ポイント行: 楽天ポイント明細セクションに「ポイント利用」行を別セクションで記載
キャッシュバック行: なし（楽天キャッシュへの還元は別途）
```

#### 5. アメリカン・エキスプレス
```
向き: 縦（A4ポートレート）
構成: Amexロゴ（青）→ アカウントサマリー（英語）→ 明細テーブル → フッター
列順: Date / Description / Amount（または 利用日 / 利用内容 / 金額）
利用店舗位置: Description列（英語・日本語混在）
日付表記: DD MMM YYYY（例: 01 JAN 2026）または YYYY/MM/DD
金額表記: カンマあり（CR表記でクレジット）
分割払い表記: ペイ・オーバー・タイム（リボ相当）としてセクション分け
リボ表記: 「ペイ・オーバー・タイム残高」として別セクション
ポイント行: メンバーシップ・リワードポイントは別紙（明細PDF内には含まれない場合多い）
キャッシュバック行: キャッシュバック型は「CASHBACK CREDIT」として負数行
外貨取引: 元通貨 + 換算レート + 円換算額の3行構成
```

#### 6. セゾンカード
```
向き: 縦（A4ポートレート）
構成: セゾンロゴ → ご請求内容 → 明細テーブル → 永久不滅ポイント残高 → フッター
列順: ご利用年月日 / ご利用内容 / 分割区分 / ご請求金額 / 今回お支払い金額
利用店舗位置: ご利用内容列
日付表記: YYYY/MM/DD
金額表記: カンマあり
分割払い表記: 分割区分列に「3回」「6回」等
リボ表記: 「リボ払い」として区分列に記載
ポイント行: 永久不滅ポイント充当は「ポイント充当」として負数行
キャッシュバック行: なし
```

#### 7. dカード
```
向き: 縦（A4ポートレート）
構成: dカードロゴ → ご請求金額 → 明細テーブル → dポイント明細 → フッター
列順: ご利用年月日 / ご利用店名・商品名 / 支払区分 / ご利用金額 / 今回のお支払い金額
利用店舗位置: ご利用店名列（NTTドコモ関連は長いサービス名）
```

## Part 4: 解析上の注意事項

### 金額解析の共通課題
- カンマ有無の混在 → 正規表現で対応: `[\d,]+` → カンマ除去してint変換
- 全角数字の混在 → 全角→半角変換が必要
- 「-」vs「△」vs「（）」のマイナス表記 → 出金/入金列の分離で対応

### 文字コード変換の注意
- Shift_JIS ファイルは `chardet` または明示的デコードで対応
- BOM付きUTF-8は `utf-8-sig` でデコード
- 半角カナはShift_JISで「ﾌﾘｺﾐ ﾏﾙﾏﾙｼﾖｳﾃﾝ」のように現れる → 全角変換推奨

### 日付解析の共通課題
- 元号表記（令和6年 = 2024年）→ 西暦変換テーブルが必要
- 年号省略（「0415」= 当該年の4月15日）→ 対象期間から年を補完
- PDF上の日付は文字列抽出でズレることがある → OCR後の正規化必要

### PDF特有の課題
- テーブル境界線の検出（pdfplumber / pdfminer 推奨）
- 2列構造（出金/入金を1行に配置）の解析は列幅でロジック分岐が必要
- スキャンPDFは画像として埋め込まれる → OCRが必要（Claude Vision / Tesseract）

### 通帳写真特有の課題
- ゆがみ補正（perspective transform）
- ドットインパクト印字の認識精度（特に薄くなった部分）
- 手書き補記（鉛筆メモ等）との区別
- 反射・グレア → 撮影時の照明指導が必要
