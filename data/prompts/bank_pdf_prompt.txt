以下の画像/PDFは{bank_name}の入出金明細書PDFです。
以下の列を抽出してJSONの配列で返してください。

【抽出列】
- date: 取引日（YYYY-MM-DD形式に統一）
- description: 摘要・取引内容
- amount_out: 出金金額（円、整数。出金がない場合はnull）
- amount_in: 入金金額（円、整数。入金がない場合はnull）
- balance: 差引残高（円、整数。記載がない場合はnull）

【フォーマット固有の注意点】
{bank_format_notes}

【出力形式】
```json
[
  {
    "date": "2026-04-01",
    "description": "取引内容",
    "amount_out": 10000,
    "amount_in": null,
    "balance": 500000
  }
]
```

【注意事項】
- 金額のカンマは除去して整数として返すこと
- 全角数字は半角に変換すること
- 日付が元号表記の場合は西暦に変換すること（令和6年=2024年、令和7年=2025年）
- 年号が省略されている場合は、明細書の対象期間から補完すること
- ヘッダー行・合計行・残高繰越行は除外すること
- 摘要が2行にわたる場合は1行に結合すること
- 口座番号や支店名など、取引でない行は除外すること
- 抽出できない行がある場合は除外してよい（エラーにしない）

# 銀行別フォーマットノート

## みずほ銀行
列順: お取引日 / 摘要 / お引き出し金額 / お預け入れ金額 / 差引残高
日付: YYYY年MM月DD日形式
金額: カンマあり
摘要列: 全角カナ表記が多い（「フリコミ タナカタロウ」等）
注意: 摘要が2行構成の場合がある。2行を結合して1レコードとする

## 三菱UFJ銀行
列順: 取引日 / 摘要 / 支払い金額 / 預かり金額 / 差引残高
日付: YYYY/MM/DD形式
金額: カンマあり（例: 1,234,567）
注意: 法人口座では「摘要2」列が追加されることがある

## 三井住友銀行
列順: 取引年月日 / 摘要 / 支払金額 / 預り金額 / 残高
日付: YYYY/MM/DD形式
金額: カンマなし
注意: 摘要列の折り返しに注意

## りそな銀行
列順: 年月日 / お取り扱い内容 / お支払い金額 / お預かり金額 / 残高
日付: YYYY/MM/DD形式
金額: カンマあり
注意: 埼玉りそな・関西みらい銀行も同形式

## 楽天銀行
列順: 取引日 / 取引内容 / 出金金額 / 入金金額 / 残高
日付: YYYY/MM/DD形式
金額: カンマあり
注意: 取引内容列にハイフン区切りの複合情報が入ることがある

## PayPay銀行
列順: 取引日 / 摘要 / 受取金額 / 支払金額 / 残高
日付: YYYY/MM/DD形式
金額: カンマなし
注意: シンプルなモノクロ帳票。受取=入金、支払=出金

## ゆうちょ銀行
列順: 年月日 / お取扱内容 / お支払金額 / お預り金額 / 差引残高
日付: YYYY年MM月DD日形式
金額: カンマあり
注意: 口座番号が「記号 XXXXX 番号 XXXXXXXX」形式。ページ末に「合計」行があるので除外する

## 住信SBIネット銀行
列順: 取引日 / 取引内容 / 出金金額 / 入金金額 / 残高
日付: YYYY/MM/DD形式
金額: カンマなし
注意: ハイブリッド預金は口座別に別PDFになっている
