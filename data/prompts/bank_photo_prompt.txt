以下の画像は{bank_name}の通帳を撮影した写真です。
通帳の明細部分を読み取り、以下の列を抽出してJSONの配列で返してください。

【抽出列】
- date: 取引日（YYYY-MM-DD形式に統一）
- description: 取引内容・摘要
- amount_out: お支払金額/お引き出し金額（円、整数。ない場合はnull）
- amount_in: お預り金額/お預け入れ金額（円、整数。ない場合はnull）
- balance: 差引残高（円、整数。ない場合はnull）

【出力形式】
```json
[
  {
    "date": "2026-04-01",
    "description": "フリコミ タナカタロウ",
    "amount_out": null,
    "amount_in": 100000,
    "balance": 500000
  }
]
```

【画像解析の注意事項】
- 通帳は横長見開き形式。列は左から「年/月日/取引内容/出金/入金/残高/取扱店番」の順が一般的
- ドットインパクト印字のため文字がかすれている場合がある。文脈から推測して補完してよい
- 年列が1桁や2桁で省略されている場合は、通帳の発行年や前後の日付から補完すること
- 元号表記（令和○年）は西暦に変換すること（令和6=2024、令和7=2025、令和8=2026）
- 全角数字・半角カナ混在に注意。数字は半角整数に統一すること
- 金額のカンマは除去して整数として返すこと
- ページ境界（「次のページへ」等）の繰越行は除外すること
- 手書きのメモ書き（鉛筆書き等）は無視してよい
- ピンボケ・反射で読み取れない行は除外してよい（エラーにしない）
- 「*」や「○」等の確認マークは無視してよい

# 銀行別 通帳フォーマットノート

## みずほ銀行
通帳カバー: 紺（個人普通）/ 茶（個人総合）/ 緑（法人）
列順: 年 / 月日 / お取引内容 / お支払金額 / お預り金額 / 差引残高 / 取扱店番
印字: ドットインパクト
特徴: 「お取引内容」列が広く2行印字がある。出金/入金列は空欄のセルがある

## 三菱UFJ銀行
通帳カバー: 赤（個人普通）/ 青（法人）
列順: 年 / 月 / 日 / 取引内容 / お支払い金額 / お預け入れ金額 / 差引残高 / 取扱店
印字: ドットインパクト
特徴: 年列が2桁（「06」=2006年）。月日は1行に収まる設計

## 三井住友銀行
通帳カバー: 緑（SMBC）
列順: 年 / 月日 / お取引内容 / お支払金額 / お預り金額 / 差引残高 / 取扱店番号
印字: ドットインパクト
特徴: 月日は「0415」のように4桁で印字

## りそな銀行
通帳カバー: 臙脂/紫（個人）/ 紺（法人）
列順: 年月日 / お取扱い内容 / お支払い金額 / お預かり金額 / 差引残高 / 取扱店番
印字: ドットインパクト
特徴: 年月日が「YYMMDD」または「YYMM/DD」形式。残高欄に「*」が印字される場合あり

## 楽天銀行
通帳カバー: 赤（楽天カラー）
列順: 取引日 / 摘要 / お引き出し / お預け入れ / 残高
印字: インクジェット or レーザー（ドットインパクトではない）
特徴: 楽天銀行は原則通帳レス。通帳印字サービス（郵便局等）利用者は少数

## ゆうちょ銀行
通帳カバー: 緑（ゆうちょ）
列順: 年月日 / お取扱内容 / お支払金額 / お預り金額 / 差引残高 / 取扱局番
印字: ドットインパクト
特徴: 年月日が「250415」（元号省略6桁）形式。全角カナ「フリコミ」「ヤクインイン」等
ページ末に「合計」行あり → 除外すること

## みずほ・三菱UFJ・三井住友 共通
- 「振込」は「フリコミ」または「ﾌﾘｺﾐ」（半角カナ）
- 「引落」は「ヒキオトシ」または「ﾋｷｵﾄｼ」
- 取引内容が「ATM」「CD」で始まる場合は現金引出

## PayPay銀行・住信SBIネット銀行
通帳なし（通帳レス銀行）。このプロンプトは使用しない
