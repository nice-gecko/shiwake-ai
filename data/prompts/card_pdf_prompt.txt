以下の画像/PDFは{card_name}のカード利用明細書PDFです。
以下の列を抽出してJSONの配列で返してください。

【抽出列】
- date: 利用日（YYYY-MM-DD形式に統一）
- merchant: 利用店名・商品名
- amount: 利用金額（円、整数）
- payment_amount: 今回のお支払い金額（円、整数。記載がない場合はamountと同値）
- payment_method: 支払区分（「一括」「分割X回」「リボ」「ボーナス一括」等）
- is_credit: クレジット/還元行かどうか（true=ポイント充当・キャッシュバック等のマイナス行）

【出力形式】
```json
[
  {
    "date": "2026-04-01",
    "merchant": "アマゾンジャパン",
    "amount": 5480,
    "payment_amount": 5480,
    "payment_method": "一括",
    "is_credit": false
  },
  {
    "date": "2026-04-01",
    "merchant": "ポイント充当",
    "amount": -500,
    "payment_amount": -500,
    "payment_method": null,
    "is_credit": true
  }
]
```

【注意事項】
- 金額のカンマは除去して整数として返すこと
- 全角数字は半角に変換すること
- ポイント充当・キャッシュバック・還元行は `is_credit: true` として含める（除外しない）
- ヘッダー行・合計行・請求金額サマリー行は除外すること
- 分割払いは `payment_method` に「分割X回」と明記し、今月分の `payment_amount` を返すこと
- リボ払いは `payment_method: "リボ"` として返すこと
- 利用者名（本人/家族）が列にある場合は `merchant` の末尾に「(家族)」等で付記してよい
- 外貨建て取引は円換算後の金額を `amount` に入れること

# カード別 フォーマットノート

## 三菱UFJカード
列順: 利用日 / 利用店名・商品名 / 利用者 / 支払方法 / 利用金額 / 当月支払額
日付: YYYY/MM/DD
金額: カンマあり
分割: 「支払方法」列に「2回払い」「3回均等分割」等
リボ: 「リボルビング払い（残高XXXXX円）」
ポイント行: 「三菱UFJカードポイント還元」として負数行 → is_credit: true

## 三井住友カード
列順: ご利用日 / ご利用店名・商品名 / お支払金額 / 今回のお支払い金額 / ご利用者名 / お支払い方法
日付: YYYY/MM/DD
金額: カンマあり
分割: 「お支払い方法」列に「分割X回」
リボ: 「マイ・ペイすリボ」。元本・手数料を分けた行あり
ポイント行: 「Vポイントサービス」として負数行 → is_credit: true
キャッシュバック行: 「キャッシュバック」として負数行 → is_credit: true

## JCBカード
列順: ご利用年月日 / ご利用店名 / お支払区分 / お支払い回数 / ご利用金額 / 割引・手数料
日付: YYYY年MM月DD日
金額: カンマあり（¥表記あり）
分割: 「お支払区分」に「分割」、「お支払い回数」に回数
リボ: 「リボ払い」。手数料は別行
ポイント行: 「OkiDoki充当」として負数行 → is_credit: true

## 楽天カード
列順: 利用日 / 利用店名・商品名 / 支払方法 / 利用金額 / 当月支払い金額
日付: YYYY/MM/DD
金額: カンマなし
分割: 「支払方法」に「分割X回払い」
リボ: 「楽Pay（リボ）」。月額固定額とは別に手数料行あり
ポイント行: 明細末尾の「楽天ポイント明細」セクションに記載 → is_credit: true
注意: 楽天市場の利用店名は長い（「楽天市場 ○○ストア」等）

## アメリカン・エキスプレス
列順: Date / Description / Amount（または 利用日 / 利用内容 / 金額）
日付: DD MMM YYYY または YYYY/MM/DD
金額: カンマあり。クレジットは「CR」表記または負数
分割: 「ペイ・オーバー・タイム」として別セクション
リボ: 「ペイ・オーバー・タイム残高」
キャッシュバック行: 「CASHBACK CREDIT」として負数 → is_credit: true
外貨取引: 元通貨・換算レート・円換算の3行 → 円換算額を `amount` に使用
注意: 英語・日本語混在の場合がある

## セゾンカード
列順: ご利用年月日 / ご利用内容 / 分割区分 / ご請求金額 / 今回お支払い金額
日付: YYYY/MM/DD
金額: カンマあり
分割: 分割区分列に「3回」「6回」等
リボ: 「リボ払い」として区分列
ポイント行: 「永久不滅ポイント充当」として負数行 → is_credit: true

## dカード
列順: ご利用年月日 / ご利用店名・商品名 / 支払区分 / ご利用金額 / 今回のお支払い金額
日付: YYYY/MM/DD
金額: カンマなし
分割: 支払区分列に「分割X回」
リボ: 「クレジット（リボ）」として区分
ポイント行: 明細末尾「dポイント充当額」セクション → is_credit: true
注意: NTTドコモ関連サービスは利用店名が長い

## イオンカード
列順: ご利用日 / ご利用先名称 / お支払区分 / ご利用金額 / 今回お支払い金額
日付: YYYY/MM/DD
金額: カンマなし
分割: お支払区分に「分割X回」
リボ: 「毎月リボ払い」として区分列
ポイント行: 「WAON充当額」として負数行 → is_credit: true
注意: イオングループの店舗名は略称が多い（「イオン」「マックスバリュ」「ミニストップ」等）
