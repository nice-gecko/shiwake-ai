## 1. 設計思想

### 1.1 既存プラン体系との位置づけ

shiwake-ai は既に**24プラン構成**で運用中(v2.2.0で確立):

| 種別 | プラン数 | 価格帯 |
|---|---|---|
| AI SaaS版(通常) | 5種 | ¥980〜¥100,000 |
| Agent版(通常) | 3種 | ¥30,000〜¥150,000 |
| Agent エリート(通常) | 1種 | ¥250,000 |
| インセンティブ(オプション) | 3種 | ¥5,000〜¥20,000 |
| 代理店プラン(卸売) | 12種 | (上記対応) |
| **合計** | **24種** | - |

この上に、**ワークスペース機能を有料化**して載せる。

### 1.2 「税理士オプション」と呼ばない理由

設計思想 v3.0 で確立した通り、shiwake-ai は「税理士限定ツール」と見られないブランディングを徹底する。

新規プロダクトの呼称は:
- ✗ 「税理士オプション」
- ✗ 「税理士プラン」
- ✗ 「Tax Office Bundle」
- ✓ **「ワークスペース10枠オプション」**
- ✓ **「追加ワークスペース10枠」**

「ワークスペース」は中立な技術語で、税理士でも個人事業主でも違和感がない。

### 1.3 なぜプレミアム以上に限定するか

ライト・スタンダードは「個人〜小規模法人」向けの価格帯。複数事業管理ニーズが薄い。

プレミアム ¥150,000 / エリート ¥250,000 は「中規模〜大規模法人」と「税理士事務所」向けの価格帯。**複数事業管理が業務要件となるユーザー層**と一致する。

これにより:
- 料金プラン体系の論理性が保たれる(Agent全体に乗らない=シンプル)
- ライト・スタンダードユーザーには余計な選択肢を見せない
- 「ワークスペース10枠オプションを使いたい→プレミアムへアップグレード」という上位プランへの誘導

### 1.4 既存代理店規約との整合

代理店規約 v1.0 draft(2026年5月8日)第5条:

> ランク別の卸売価格および代理店マージンは、次のとおりです。
> | Bronze | 〜¥100,000 | 70% | 30% |
> | Silver | ¥100,001〜¥500,000 | 65% | 35% |
> | Gold | ¥500,001〜 | 60% | 40% |

新規2プロダクトもこの階段制に統合。**取引高判定の対象に含める**(Bronze 70%卸売額で集計)。これにより:
- 規約改定が最小限で済む(プラン定義表に行を追加するのみ)
- 代理店から見て「同じルール」で運用できる
- 兄(代理店候補)への説明がシンプル

## 2. 新規プロダクトの仕様

### 2.1 プロダクト1: ワークスペース10枠オプション

#### 概要

Agent プレミアム または Agent エリート に対する**追加サブスクリプション**。
契約者のワークスペース上限を **1個 → 10個**に解放する。

#### 詳細仕様

| 項目 | 内容 |
|---|---|
| **plan_key** | `workspace_option_10` |
| **edition** | `option` |
| **商品名** | ワークスペース10枠オプション |
| **月額(通常価格)** | ¥20,000 / 月 |
| **月額(Bronze卸売)** | ¥14,000 / 月 |
| **解放するワークスペース数** | 10個 |
| **対応プラン** | Agent プレミアム、Agent エリート(代理店プラン含む) |
| **重複加入** | 不可(同一ユーザーで複数加入不可) |
| **加入時の即時解放** | 解放(契約日からワークスペース10個まで作成可能) |
| **解約時の挙動** | 11個目以降のワークスペースは閲覧のみ可能(`A-3a §8.3` 参照) |

#### Stripe price_id(本番モード作成時)

```
price_workspace_option_10_normal   = ¥20,000/月
price_workspace_option_10_wholesale = ¥14,000/月(代理店用)
```

### 2.2 プロダクト2: 追加ワークスペース10枠

#### 概要

「ワークスペース10枠オプション」加入者が、さらにワークスペースを追加するためのサブスクリプション。
**10個単位で追加可能**、購入数に上限なし。

#### 詳細仕様

| 項目 | 内容 |
|---|---|
| **plan_key** | `workspace_addon_10` |
| **edition** | `option` |
| **商品名** | 追加ワークスペース10枠 |
| **月額(通常価格)** | ¥10,000 / 月 |
| **月額(Bronze卸売)** | ¥7,000 / 月 |
| **解放するワークスペース数** | +10個(累積) |
| **加入の前提** | `workspace_option_10` への加入が必要 |
| **重複加入** | **可能**(購入数だけワークスペース数が増える) |
| **解約時の挙動** | 解約後、上限超過分のワークスペースは閲覧のみ可能 |

#### Stripe price_id(本番モード作成時)

```
price_workspace_addon_10_normal   = ¥10,000/月
price_workspace_addon_10_wholesale = ¥7,000/月(代理店用)
```

#### 累積購入の例

| 加入状況 | ワークスペース上限 | 月額(通常価格) |
|---|---|---|
| プレミアム単体 | 1個 | ¥150,000 |
| プレミアム + 10枠オプション | 10個 | ¥170,000 |
| プレミアム + 10枠オプション + 追加10枠×1 | 20個 | ¥180,000 |
| プレミアム + 10枠オプション + 追加10枠×3 | 40個 | ¥200,000 |
| プレミアム + 10枠オプション + 追加10枠×5 | 60個 | ¥220,000 |

### 2.3 エリートに含めるべきか?

設計選択肢:
- **案A**: エリートには `workspace_option_10` を**標準装備**(オプション加入不要で最初から10枠)
- **案B**: エリートも別途オプション加入が必要

#### 推奨: 案A(エリート標準装備)

理由:
- エリート ¥250,000 の差別化価値を高める(対話モード+ワークスペース込み)
- 兄に見せる時の説明がシンプル「エリートはワークスペース10個込み」
- A-3a §8.1 のプラン制約表とも整合

#### 実装方針(案A の場合)

```javascript
// プラン判定時にワークスペース上限を返す
function getWorkspaceLimit(planKey, hasOptionTen, addonTenCount) {
  if (planKey === 'agent_elite' || planKey === 'reseller_agent_elite') {
    // エリートは10個込み + addon10×n
    return 10 + (addonTenCount * 10);
  }
  if (hasOptionTen) {
    // プレミアム + オプション = 10個 + addon10×n
    return 10 + (addonTenCount * 10);
  }
  // それ以外は1個
  return 1;
}
```

エリートユーザーは `workspace_option_10` を購入せずに10個使えるが、それ以上欲しければ `workspace_addon_10` を購入。

## 4. 代理店制度との統合

### 4.1 マージン構造(既存階段制を流用)

ワークスペース機能の2プロダクトに対する代理店マージン:

| ランク | 取引高条件 | ワークスペース10枠オプション | 追加ワークスペース10枠 |
|---|---|---|---|
| Bronze | 〜¥100,000/月 | 卸¥14,000 / マージン¥6,000(30%) | 卸¥7,000 / マージン¥3,000(30%) |
| Silver | ¥100,001〜¥500,000/月 | 卸¥13,000 / マージン¥7,000(35%) | 卸¥6,500 / マージン¥3,500(35%) |
| Gold | ¥500,001〜/月 | 卸¥12,000 / マージン¥8,000(40%) | 卸¥6,000 / マージン¥4,000(40%) |

#### Stripe Coupon による動的価格

既存設計と同様、**代理店は常に Bronze 価格で契約**し、Silver/Gold ランクの場合は Coupon で割引適用:

```javascript
// 代理店ランクごとの Coupon(既存)
const SILVER_COUPON_ID = 'silver_discount';  // 7.14% off → 実質65%価格
const GOLD_COUPON_ID = 'gold_discount';      // 14.28% off → 実質60%価格
```

これにより:
- 既存の Coupon ロジック(`shiwake-ai_v2_2_0_仕様書.md §6.3` 参照)が新プロダクトにも自動適用される
- 新規Couponの作成は不要
- 月次バッチでの自動ランク判定もそのまま動く

### 4.2 取引高判定への組み込み

代理店規約第6条第4項:
> 取引高の判定基準は、ランクに関わらず**Bronze価格(70%)で計算した卸売価格の合計額**とします。

新プロダクトもこの判定対象に含める:

```javascript
// 月次取引高計算ロジック(server.js のランク判定バッチに追加)
async function calculateMonthlyVolume(resellerUid) {
  const subscriptions = await getResellerSubscriptions(resellerUid);
  let totalVolume = 0;
  for (const sub of subscriptions) {
    // Bronze価格で計算(70%卸価格)
    const bronzePrice = WHOLESALE_PRICES_BRONZE[sub.plan_key];
    totalVolume += bronzePrice;
  }
  return totalVolume;
}

// 例: 代理店が顧客に以下を販売した場合
// - Agent プレミアム × 5社 = ¥105,000 × 5 = ¥525,000
// - ワークスペース10枠オプション × 5社 = ¥14,000 × 5 = ¥70,000
// - 追加ワークスペース10枠 × 3社 = ¥7,000 × 3 = ¥21,000
// 合計 ¥616,000 → Gold ランク
```

### 4.3 代理店規約改定の最小化

第5条(卸売価格・代理店マージン)第2項の表に2行を追加する想定。**他の条文は変更なし**:

```markdown
| ランク | 月次取引高 | 卸売率 | 代理店マージン |
|---|---|---|---|
| Bronze | 〜¥100,000 | 70% | 30% |
| Silver | ¥100,001〜¥500,000 | 65% | 35% |
| Gold | ¥500,001〜 | 60% | 40% |

【追記】上記マージン率は、以下のプロダクトを含む全代理店プランに適用されます:
- AI SaaS版 全プラン
- Agent版 全プラン
- インセンティブオプション
- ワークスペース10枠オプション
- 追加ワークスペース10枠
```

これにより**規約全体の構造を保ったまま新機能を組み込み**できる。

### 6.2 ワークスペース上限の計算

```javascript
// server.js: ワークスペース上限計算関数
async function getWorkspaceLimit(uid) {
  const user = await supabaseQuery(
    `/users?id=eq.${uid}&select=plan_key,has_workspace_option,workspace_addon_count`
  );
  if (!user[0]) return 0;

  const { plan_key, has_workspace_option, workspace_addon_count } = user[0];

  // エリートは10枠標準装備
  const isElite = plan_key === 'agent_elite' || plan_key === 'reseller_agent_elite';
  const baseLimit = (isElite || has_workspace_option) ? 10 : 1;

  // 追加枠を加算
  const addon = (workspace_addon_count || 0) * 10;

  return baseLimit + addon;
}
```

### 6.3 Stripe Webhook の対応

`/api/stripe/webhook`(`server.js:1381`)に、新プロダクト購入時のフラグ更新ロジックを追加:

```javascript
// invoice.paid イベント時の処理
if (event.type === 'invoice.paid') {
  const subscription = event.data.object;
  const priceId = subscription.items.data[0].price.id;
  const uid = await getUidFromCustomerId(subscription.customer);

  // ワークスペース10枠オプションの加入
  if (priceId === PRICE_WORKSPACE_OPTION_10_NORMAL ||
      priceId === PRICE_WORKSPACE_OPTION_10_WHOLESALE) {
    await supabaseQuery(`/users?id=eq.${uid}`, 'PATCH', {
      has_workspace_option: true
    });
  }

  // 追加10枠の購入
  if (priceId === PRICE_WORKSPACE_ADDON_10_NORMAL ||
      priceId === PRICE_WORKSPACE_ADDON_10_WHOLESALE) {
    // 既存購入数を +1
    const current = await supabaseQuery(
      `/users?id=eq.${uid}&select=workspace_addon_count`
    );
    await supabaseQuery(`/users?id=eq.${uid}`, 'PATCH', {
      workspace_addon_count: (current[0]?.workspace_addon_count || 0) + 1
    });
  }
}

// subscription.deleted イベント時の処理(解約時)
if (event.type === 'customer.subscription.deleted') {
  const subscription = event.data.object;
  const priceId = subscription.items.data[0].price.id;
  const uid = await getUidFromCustomerId(subscription.customer);

  // ワークスペース10枠オプションの解約
  if (priceId === PRICE_WORKSPACE_OPTION_10_NORMAL ||
      priceId === PRICE_WORKSPACE_OPTION_10_WHOLESALE) {
    await supabaseQuery(`/users?id=eq.${uid}`, 'PATCH', {
      has_workspace_option: false
    });
  }

  // 追加10枠の解約 → 購入数を -1
  if (priceId === PRICE_WORKSPACE_ADDON_10_NORMAL ||
      priceId === PRICE_WORKSPACE_ADDON_10_WHOLESALE) {
    const current = await supabaseQuery(
      `/users?id=eq.${uid}&select=workspace_addon_count`
    );
    const newCount = Math.max(0, (current[0]?.workspace_addon_count || 0) - 1);
    await supabaseQuery(`/users?id=eq.${uid}`, 'PATCH', {
      workspace_addon_count: newCount
    });
  }
}
```

⚠️ **重要**: 追加10枠の解約は「最後に加入したサブスクリプション1つの解約」を意味する。複数加入している場合、解約のたびに `workspace_addon_count` が `-1` される。

### 6.4 解約後の挙動(上限超過対応)

ユーザーが上限超過状態になった場合(`現ワークスペース数 > 上限`):
- 全ワークスペースは**閲覧のみ可能**
- 編集・新規仕訳は不可
- 警告バナーを表示(§5.2)
- ユーザーがアーカイブを選ぶ → 上限以下に戻ったら通常動作復帰

```javascript
// POST /api/workspaces
async function createWorkspace(req, res) {
  const { uid, name } = req.body;

  // 上限チェック
  const limit = await getWorkspaceLimit(uid);
  const current = await supabaseQuery(
    `/workspaces?owner_uid=eq.${uid}&is_archived=eq.false&select=count`
  );
  const currentCount = current[0]?.count || 0;

  if (currentCount >= limit) {
    res.writeHead(400, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({
      error: 'workspace_limit_exceeded',
      current_count: currentCount,
      limit: limit,
      upgrade_url: '/pricing#workspace-option'
    }));
    return;
  }

  // 通常の作成処理
  // ...
}
```

## 10. Claude Code 向け実装指示

### 10.1 実装の順序

詳細は §9.2 のフェーズ分割を参照。

### 10.2 注意事項

#### 重要1: Stripe本番モード移行と同時実施が効率的
v2.3.0 で予定されている本番モード移行と並行して進めると、Stripe Dashboard 作業が1回で済む。

#### 重要2: webhook の冪等性
`workspace_addon_count` のインクリメント処理は、**同じ webhook が複数回呼ばれても重複加算しないよう冪等性を担保**する必要がある:
- Stripe の `idempotency_key` を活用
- または、subscription_id 単位で増減を記録(`workspace_addon_subscriptions` テーブル新設も検討)

**推奨実装**:
```sql
CREATE TABLE workspace_addon_subscriptions (
  subscription_id TEXT PRIMARY KEY,
  uid TEXT NOT NULL,
  status TEXT NOT NULL,  -- 'active' or 'cancelled'
  created_at TIMESTAMPTZ DEFAULT now()
);
```

これで「現在 active な addon の数 = workspace_addon_count」と確実に同期できる。

#### 重要3: エリートユーザーのオプション加入を防ぐ
エリートは標準装備で10枠あるため、`workspace_option_10` への加入は不要。
Stripe Checkout 生成時にチェック:
```javascript
if (planKey === 'workspace_option_10' && currentUser.plan_key === 'agent_elite') {
  return error('エリートプランには10枠が標準装備されています');
}
```
