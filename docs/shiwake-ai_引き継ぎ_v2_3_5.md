## ⚠️ 重要な技術的注意点(本セッション学び)

### CC の push 忘れが3回発生
- Group 1 完了後、Group 2 完了後、Group 4 完了後の3回
- いずれも「完了報告」した後に git log origin/main で発覚
- CLAUDE.md に記載済みのルール違反

**再発防止策**(次セッション以降の CC 依頼文に必須):
- 「完了報告に必ず `git log origin/main -1` の出力を含めること」を明記する
- 「完了の定義は push origin main まで完走したこと」と毎回明記する

### NOT NULL カラム漏れの連続発生(2回)
- inbox_files テーブルの実スキーマを CC が確認しきれず、テストデータINSERTが2回エラー
- 「server.js の3箇所のINSERT を実スキーマ代わりに使う」アプローチで最終的に解決

**再発防止策**:
- 既存テーブルへのINSERT文を作る際は、`information_schema.columns` を WHERE table_name で参照する SQL を先に出力させて、スキーマ確認してから INSERT 文生成

### 表示制御関数の呼び出しタイミング(レースコンディション再現)
- setupAutoIntakeMenuItem() は実装されていたが onAuthStateChanged から呼ばれず DOMContentLoaded タイマー任せ
- v2.3.2 で確立した「認証完了 + WS取得完了 ゲートで全 fetch を待たせる」原則の派生版
- 既存パターンを CC が見落とすケースがあるので、新規メニュー追加時は onAuthStateChanged 経由を明示する
