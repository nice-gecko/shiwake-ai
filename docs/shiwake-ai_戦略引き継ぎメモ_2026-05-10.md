## 3. 戦略議論の結論

### 3.1 ターゲットのピボット

**個人向け直販 → 税理士代理店モデル(B2B2B)** に方針転換。

理由:
- 個人/SMB向けは freee、マネフォと正面衝突
- ユーザーの実兄が税理士で、既にFBをくれている
- 税理士1人 = 顧問先10〜100社 → 集客が一気に解決
- マネフォ「公認メンバー」、freee「認定アドバイザー」、弥生「PAP」など、**業界モデルとして実証済み**
- 税理士は仕訳の自動化を顧問先に勧めたいが、既存ソフトを強制できない悩みあり

### 3.2 設計思想の見直し

**v2.0.0 で設計された「キャリアパス・成長物語」演出を、税理士B2B向けに業務語彙へリブランド。**

ユーザー自身も成長ストーリー演出に違和感を持っていた。兄からのFBは「**第一に信頼性**」。50件の慣らし運転で精度を見極めてから自動化解放、という流れ自体は維持(設計として合理的)。

#### 言葉の差し替え方針(機能はそのまま、UI語彙のみ変更)

| 今(成長物語) | 提案(信頼検証) |
|---|---|
| ルーキー卒業 | 慣らし運転完了 |
| ジュニアに昇格 | 自動取り込みを解放 |
| 育成中 ⭐⭐⭐⭐⭐ | 学習済み 79件 / 精度97% |
| あなたは成長しました | AIの精度をご確認ください |
| 🎉 卒業おめでとう | ✓ 自動化の準備が整いました |

### 3.3 「精度」を可視化する仕組みの追加

50件処理しただけでは「信頼性検証」にならない。**何をもって信頼するかを数字で見せる**必要がある。

追加するメトリクス案:
- **承認率**: 「AIの提案をそのまま承認: 92件 / 修正: 8件 = 精度92%」
- **修正パターン**: 「最も修正が多かった: 勘定科目(5件)」
- **学習済み取引先**: 「AIが覚えた取引先: 23社」
- **慣らし運転完了の判断材料**: 「精度95%以上で自動化推奨」

これにより、ユーザー(または税理士)が**根拠を持って自動化解放できる**。

## 【将来のビジネス展開メモ - 2026-05-11 着想】

shiwake-ai が運用フェーズに入ると、ユーザー全体の仕訳データが蓄積される。これを二次利用したビジネス展開の方向性を検討する価値あり(規約整備前提)。

### 想定される方向性

| 方向性 | 概要 | 優先度 | リスク |
|---|---|---|---|
| A. 個社向け分析 | 顧問先ごとに「あなたの会社の経費構造」レポート提供 | 高 | 規約整備で対応可、最も無難 |
| B. 業界ベンチマーク | 匿名化集計データを業種別に比較提供 | 中 | 個社特定リスク、最小サンプル数設計必要 |
| C. マクロ経済データ販売 | 業種別の平均仕入率等を金融機関等に提供 | 中 | 法的論点(個情法、金商法) |
| D. AI学習データ販売 | 仕訳データセットを他社AIに販売 | 低 | ユーザー反発リスク大 |
| E. 与信・スコアリング | 仕訳の健全性から事業者評価 | 低 | 金商法・貸金業法、極めて慎重に |

### 検討メモ
- **A(個社分析)から始めるのが筋**(自社ユーザーへの還元、規約も最小限で済む、信頼度を損なわない)
- B 以降は「データ提供への明示的同意」を利用規約に組み込んでから検討
- 全方向性に共通して、匿名化処理・k-匿名性の確保が必須
- 税理士の倫理基準(税理士法、職業倫理)に抵触しないか要検討
- 法務レビューが必須(個人情報保護法、特定商取引法)

### 次のアクション(将来)
- v2.5.x で代理店向けの「ベンチマーク機能」として A を実装検討
- 規約改定: 「匿名化された統計データの二次利用に同意する」条項追加(同意取得)
