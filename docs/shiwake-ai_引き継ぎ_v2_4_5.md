## 3. アンロック機能一覧画面 — 最終仕様(コミット 5d24bc3 → 0619a12 → c79d3a0)

### 3.1 配置
- サイドバー「未振り分け」(`navUnassignedTray`)の直後
- サイドバー「管理者切替」(`adminViewToggle`)の直前

### 3.2 ヘッダー
- タイトル: 「アンロック機能」
- サブテキスト: 「使うほど、自分専用のツールに育っていきます」(設計思想を体現)

### 3.3 カード構成(3カード)

| 順 | カード名 | 説明 | プラン | 状態判定 | アクション |
|---|---|---|---|---|---|
| 1 | 自動インポート | メール / Dropbox / Google Drive から証憑を自動で取り込みます | 全プラン | `window._workspaces` の当該WSの `client_email_addresses.length > 0 OR client_email_domains.length > 0 OR subject_keywords.length > 0` | 「設定を開く」→ `switchTab('auto-intake')` |
| 2 | 自動承認(将来実装) | エージェントモードで自動仕訳・自動承認を実行する次世代機能(将来実装予定) | Agent版以上 | 常時「将来実装」プレースホルダー | 「プランを比較」→ `openUpgradeModal()` |
| 3 | 自動エクスポート | 承認済み仕訳が一定件数溜まったら、メール / クラウド / ローカルへ自動でCSVを送ります | 全プラン | `/api/auto-export/settings` を fetch → `auto_export_enabled` | 「設定を変更」→ `openWsEditAndScroll()` で WS編集モーダル + `auto-export-section` へスクロール |

### 3.4 ON/OFFトグル機能(自動エクスポートのみ)
- iOS風スイッチデザイン(40px × 22px、青塗り `#185FA5` ON / グレー OFF)
- クリックで `PATCH /api/auto-export/settings` を即時実行(`{uid, workspace_id, enabled: bool}`)
- 自動インポートは設定が複雑(複数フィールド)のためトグル化せず、「設定を開く」誘導のみ
- 自動承認はトグルなし(将来実装)

### 3.5 スクロールジャンプ実装
- `openWsEditAndScroll()` 関数を新規追加(既存 `openWsEdit(wsId)` をそのまま呼び出し + 600ms 後に `scrollIntoView({behavior:'smooth', block:'start'})`)
- 既存関数は無改造、wrap で対応

### 3.6 将来追加予定の機能(8カード構想)

現状の3カードに加え、将来5カードが追加される設計:

| 将来追加 | プラン | 実装予定 |
|---|---|---|
| 自動ルール学習(Phase 3) | Agent版以上 | v2.5.0 |
| 対話モード(Slack/Teams/LINE/音声) | **Elite限定** | v3.1.0〜v3.2.0 |
| 文脈横断判断 | **Elite限定** | v3.3.0 |
| 締め自走 | **Elite限定** | v3.4.0 |
| 決算・税務の事前準備 | **Elite限定** | v3.5.0 |

→ 最終形は 8カード(現状3 + 将来5)。Elite アップグレード訴求の核となる予定。

## 4. 全UI用語統一 — 最終形(コミット 0619a12 → c79d3a0)

### 4.1 経緯(重要な意思決定)

1. **第1案(0619a12)**: 漢字四文字統一「自動取込 / 自動承認 / 自動書出」
   - 設計思想「専門性の強化」と整合

3. **第2案(c79d3a0、最終確定)**: カタカナ語に戻す「自動インポート / 自動承認 / 自動エクスポート」
   - 瞬間理解性を優先(UX > 専門性)

### 4.2 最終的な用語ルール

| 領域 | 表記 |
|---|---|
| 機能カード・サイドバー・WSモーダル等の全UI | 「自動インポート」「自動エクスポート」(カナ語) |
| エージェントモード関連 | 「エージェント」(カタカナ統一、英語「Agent」廃止) |
| Stripe プラン名(`agent_lite` 等の内部ID) | 維持(Stripe ID 連動のため触らない) |

### 4.3 統一できなかった箇所(意図的な維持)

| 箇所 | 理由 |
|---|---|
| JS コメント内「自動取り込み」(5360/5511/5693/6199/6219行) | UIテキストではない |
| CSS コメント内「自動エクスポート設定」(120行) | 同上 |
| プラン名「Agent ライト/スタンダード/プレミアム/エリート」 | Stripe ID と紐付くため維持 |
| 会計ソフト取込ガイド内の一般動詞「取込」5件(5283/5305/5316等) | 「会計ソフトへのCSV取り込み操作」の文脈で機能名ではない |
