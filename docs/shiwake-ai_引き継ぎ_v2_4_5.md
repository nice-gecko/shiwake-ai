# 証憑仕訳AI 引き継ぎドキュメント

URL: https://shiwake-ai.onrender.com/

---

## プロダクト概要

領収書・レシート・通帳の画像やPDFをアップロードすると、弥生会計用の仕訳CSVを自動生成するAIツール。

**ターゲット：** 個人事業主・中小企業の経理担当者・税理士  
**守備範囲：** 証憑のアップロード〜CSV出力まで（会計ソフト側には手を出さない設計）  
**対応会計ソフト：** 弥生会計（CSV形式でインポート）

---

## 現在の主な機能

- 証憑アップロード（JPG / PNG / PDF、最大4枚）
- AIによる仕訳自動生成（勘定科目・税区分・摘要）
- AIの根拠テキスト表示（例：「Apple関連サービスは通信費に分類」）
- 信頼度バッジ表示（高信頼 / 要確認など）
- 承認 / 修正 / 削除ボタン
- 取引先マスタ（承認時に自動登録・次回以降の仕訳に即時反映）
- CSV出力（承認済みのみ or 全件）

---

## 重要な設計思想

**使えば使うほど賢くなる学習ループが実装済み**

```
証憑の文字列 → AIが推定 → 人間が修正して承認
→「この文字列 = この勘定科目・取引先」として記憶
→ 次回は最初から正しく出る
→ 同一セッション内でも承認後の仕訳に即時反映
```

これが核心機能。取引先マスタが育つほど、カタカナ途中切れのクレカ明細などへの対応精度が上がる。

## UI改善タスク一覧

### 🔴 優先度高

**① ファーストビューにキャッチコピーを追加**
- アップロードエリアの上に何もなく、初見ユーザーが何のサービスか3秒で理解できない
- 案：`領収書を投げ込むだけで、弥生用の仕訳CSVが出てくる`

**② 取引先マスタの重複レコード対処**
- 「カブシキガイシャループ」が2行登録されている
- 表記ゆれ吸収（全角/半角、カタカナの部分一致など）の強化が必要
- マスタが汚れると精度劣化につながるため早急に対処

**③ 「全件クリア」ボタンに確認ダイアログを追加**
- 現状は誤タップ一発でマスタが全消えする
- 必ず「本当に削除しますか？」の確認を挟む
- 将来的にはエクスポート／インポート機能に発展させると良い

### 🟡 優先度中

**④ ステップ進捗バーをヘッダー下に設置**
- ユーザーが今どのステップにいるかを常に把握できるようにする
- 案：`● アップロード → ○ AI処理中 → ○ 確認・承認 → ○ CSV出力`

**⑤ 処理中の件数表示**
- 現状：`PDFを分割中...`
- 改善：`PDFを分割中...（全12件を処理予定）`
- 件数が見えるだけで「ちゃんと動いてる」感が出る

**⑥ 「仕訳を自動生成する」ボタンの誘導テキスト**
- アップロード前はグレーアウトしているが、なぜ押せないか初見では不明
- ボタン下に小さく「↑ まず証憑をアップロードしてください」を追加

**⑦ 承認ボタンのサイズ見直し**
- 現状：承認ボタンが横幅いっぱいで、修正・削除が隅に追いやられている
- 改善：3ボタンを横並びで同等の存在感に（誤タップ防止）

### 🟢 優先度低（あると良い）

**⑧ 仕訳カードにサムネイル表示**
- 現状：左上がグレーの汎用アイコン
- 改善：アップロードした証憑の画像サムネイルを小さく表示
- 「どの領収書の仕訳か」が一目でわかる

**⑨ 借方・貸方の色設計の見直し**
- 現状：借方=青、貸方=赤（「未払金」が赤＝エラーに見える）
- 改善案：借方・貸方ともに黒かグレーに統一し、勘定科目名を大きく表示

---

## 差別化ポイント（外部に伝えるべき強み）

1. **「AIの根拠」テキスト表示** → 承認ボタンを押す心理的ハードルを下げる
2. **信頼度バッジ** → ユーザーが確認の優先順位をつけられる
3. **使うたびに育つ取引先マスタ** → 個人最適化の資産が蓄積される
4. **CSV出力までに絞った設計** → 既存会計ソフトと戦わない・導入ハードルが低い
