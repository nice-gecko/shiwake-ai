### ロードマップ再編
- v3.0.0「真のAgent化」を廃止 → v3.x = Elite に統合
- 理由: 能動的質問はWebアプリ内では不可能、Slack/LINE連携 = Elite が担う
- プレゼンゴール = v2.10.0 まで（Eliteはビジョンとして提示）

### プレゼン戦略
- 兄（税理士）への訴求軸: 導入ハードル低 / 精度向上 / スピード
- スタッフは新しいことを嫌う → 「作業は変わらない、AIが裏で働く」がストーリー
- Elite は行動変容の壁の先にある → 実績を積んでから提案

## v2.7.0 突合機能の現状

### 完了済み
- **DB**: 3テーブル（reconciliation_sources / reconciliation_entries / partner_aliases）＋RLS
- **API**: 11エンドポイント（CRUD / CSVアップロード / 自動マッチング / エイリアス管理 / matched-records）
- **フロントUI**: 左右2カラム構造（A通帳 vs B照合先）
  - B列: 照合済み=✓詳細表示 / 未照合=証憑追加or対応不要
  - Lv1/Lv2 とも自動照合（candidateは廃止、全部matched）
  - 照合済み行に取り消しアイコン → 確認モーダル（照合し直す or 外す）
  - 未照合行: 「証憑を追加」「対応不要」の2アクション
  - 取引先登録モーダル（通帳表記のプリフィル付き）
  - 差額理由モーダル / 対応不要モーダル
- **テスト**: sample_bank_match_test.csv で自動マッチング動作確認済み（2件照合、5件未照合）
- **データ**: CSVフォーマット定義16社 + PDF/写真フォーマット情報 + Sonnet解析プロンプト

### 次にやること
1. **CSVフォーマット定義のパーサー組み込み** — csv_formats.json をserver.jsのパーサーで使う
2. **PDF/写真アップロード対応** — Haiku分類→Sonnet解析パイプライン実装
3. **1:N マッチング** — カード引落の合計照合
4. **N:1 マッチング** — まとめ払いの組み合わせ照合
5. **欠落検知** — 定期支払いの未計上アラート
6. **UI微調整** — 照合済み行の取引先登録ボタン非表示等
