## 🎯 戦略の核 - なぜ Elite を新設するのか

### 1. 価格天井をさらに引き上げる

v2.0.0 時点で Agent プレミアム ¥200,000/月 が最上位だったが、これは「ツール」と比較されない範囲ではあるものの、**人件費比較では中途半端**だった。

Elite を ¥250,000/月 に置くことで:
- 「スタッフ1人(月給25〜35万)雇うより**確実に**安い」と説明できる
- 24時間動く・辞めない・対話できるという付加価値で**人件費比較の優位性が完成**する
- LP上で「**月¥25万のエリートエージェント**」というフレーズが成立する

### 2. 差別化機能を1箇所に集約する

対話モード、文脈横断判断、締め自走、決算事前準備 ── これらを Agent プレミアム以下に分散させると、各プランの境界がぼやける。Elite に集約することで:
- **Eliteの特別感**が明確になる
- 既存プランの構造を崩さず追加できる
- 「Eliteを目指す」というキャリアパスの頂点が物語として機能する

### 3. セルフ差別化の完成

v2.0.0 で確立した「AI SaaS版 ↔ Agent版」の2段階セルフ差別化に、**Agent内部の3段階差別化**を加える。
これにより**3層構造の比較**が成立し、各プランの選択理由が明確になる。

```
SaaS版(¥980〜¥100,000)        ← 自分で操作する
   ↓ アップセル
Agent版(¥30,000〜¥150,000)     ← 自走する
   ↓ アップセル
Agent Elite(¥250,000)           ← 対話する同僚AI
```

### 4. 内向き呼称「ダブルO」が開発文化を作る

外向き = Agent Elite(分かりやすく、営業現場で説明しやすい)
内向き = ダブルO / Double-O(007由来、開発・社内ドキュメント・slackでの会話)

この分離は AppleのiPhoneコードネーム(N81等)、Googleのプロジェクトコードネームと同じ流儀。**開発者が遊び心を持てる反面、外向きには真面目で売りやすい**。

### なぜ Agent Elite が成立するのか

- 比較対象は「ツール」ではなく「**人**」
- 月給25〜35万円のスタッフを1人雇うより、確実に安い
- スタッフは1人月160時間しか働かないが、Elite は **24時間×7日**動く
- スタッフは退職する、Elite は退職しない
- スタッフは複数チャネル(画面・slack・LINE・音声)を同時に扱えないが、Elite はできる
- スタッフは離職率が業界平均20%超だが、Elite は契約解除しない限り続く
- これらの優位性を**¥25万円**で買える、という物語

## 🎬 Agent Elite 専用機能(4能力の定義)

Lv4(エージェント)までは v2.0.0 で定義済み。Lv5(エリート)で**追加**される4能力を定義する。

### 能力1: 対話モード(コア機能)

> 「**画面の外で動く Agent**」

#### 対応チャネル
- **Slack** / **Microsoft Teams**(企業向け、優先実装)
- **LINE**(中小事業者向け)
- **音声**(STT経由、Slack音声メッセージ等の活用)
- **メール**(従来チャネルとして併存)

#### 主な動作シナリオ

**朝のサマリー通知(プッシュ)**
> 「おはようございます。昨日の証憑15件処理しました、3件だけ判断ほしいです:
> ① ◯◯商店 ¥3,200(会議費 or 接待交際費?)
> ② Amazon ¥48,000(消耗品費 or 工具器具備品?)
> ③ ◯◯ホテル ¥12,000(出張旅費 or 接待交際費?)
> 上から順に、ボタンで返信してください。」

**判断保留の対話解決(プル)**
> ユーザー: 「これ会議費でいい?」
> Agent: 「直近30日で同様の取引が3件、すべて会議費で承認されています。会議費で確定しますか?」
> ユーザー: 「OK」
> Agent: 「✅ 確定しました。今後同様のパターンは自動承認候補に追加します。」

**移動中の音声指示(音声入力)**
> ユーザー(音声): 「あれ交際費じゃなくて会議費ね」
> Agent: 「直近に保留中の◯◯商店 ¥3,200のことでしょうか? 会議費に変更しますか?」
> ユーザー: 「うん」
> Agent: 「✅ 変更しました。」

**承認スタックの督促(プッシュ)**
> 「承認スタック12件、田中さん担当が7件、最古3日前です。明後日が月次締めなので、本日中の処理を推奨します。」

#### 設計上の難所と対応方針

- **マルチチャネル状態同期**: Slack で承認した件が、Web画面でも即座に承認済み表示になる必要がある。イベントソーシング的に1つのタスクキューを真実の源とする設計。
- **監査ログ**: 誰が・いつ・どの媒体で・何を承認したか、すべて記録。J-SOX対応。音声入力の場合は STT結果テキストとセットで保存。
- **通知頻度のチューニング**: デフォルトは「朝1回サマリー、夕方1回締め前リマインド、緊急のみ即時」。ユーザーが調整可能。
- **曖昧な指示の解釈**: 「あれ」「これ」を直近の通知文脈+処理中キュー+過去パターンから推論。間違えるくらいなら「これですか?」と聞き返す保守的な設計。

### 能力2: 文脈横断判断

> 「**目の前の証憑だけを見ない Agent**」

Lv4 までは「目の前の証憑1枚」を処理する。Lv5 は**複数の証憑・契約書・過去仕訳を横断**して文脈を読む。

#### 動作例
- 「先月の交通費精算と整合性が取れません(同じ日に新幹線と航空券が重複計上)」
- 「この支出、◯◯案件のプロジェクトコードが付いてないようです。前回までは付いていました」
- 「契約書では月¥50,000の固定費なのに、今月は¥75,000で請求が来ています。確認が必要かもしれません」
- 「過去6ヶ月、◯◯商店からの請求はすべて月末でしたが、今回は月中です」

これは**経理担当者の認知負荷を肩代わりする**機能。人間が気づきにくいパターンを能動的に指摘する。

### 能力3: 締め自走

> 「**月次締めを諦めない Agent**」

月次締めのスケジュールを Agent 側で管理し、締めまで自動でリマインドし続ける。

#### 動作例
- 月末7日前: 「あと7日で締めです。現在残9件、内訳:田中さん5件、佐藤さん4件」
- 月末3日前: 「あと3日。田中さんの5件はすべて旅費精算で、領収書添付待ちです」
- 月末1日前: 「明日が締めです。最後の1件は◯◯案件の支払い、承認者は山田部長です」
- 月末当日: 「本日締めです。残り2件、いずれも田中さん。先方に再度通知しますか?」

しつこい(=信頼できる)同僚AI として機能する。締めが終わるまで諦めない設計。

### 能力4: 決算・税務の事前準備

> 「**年間カレンダーを忘れない Agent**」

決算月の3か月前から、季節性のあるタスクを能動的に動き始める。

#### 動作例
- 決算3か月前: 「そろそろ◯◯の処理を確認しましょう。前期は◯月にやっています」
- 決算2か月前: 「未払金・前払費用の見直し時期です。前期実績を添付します」
- 決算1か月前: 「棚卸チェックの依頼を顧問先に送付する時期です。テンプレート用意しました」
- 法定調書時期: 「12月になりました。法定調書の準備を始めますか?」
- 償却資産税時期: 「1月末申告の償却資産税、対象資産リストを準備しました」

年間業務カレンダーを内蔵し、税理士事務所の年間サイクルに沿った提案を能動的に行う。

## 🎯 ターゲット顧客像(Elite版)

### Elite のコアターゲット

スタッフ数20〜50名規模の税理士事務所/税理士法人で、以下の特徴を持つところ:

- 記帳代行のボリュームが特に大きい(月3,000件以上)
- 離職率が経営課題として認識されている
- スタッフ採用コスト(求人広告・教育コスト)が年間数百万円規模
- 顧問先からの「月次締めが遅い」というクレームに悩んでいる
- 経営者が「人月の神話」から脱したいと考えている
- すでに Agent プレミアムを使っていて、さらに上を求めるケース

### Eliteへのアップセル導線

```
Agent ベーシック導入 (¥30,000)
   ↓ 3ヶ月使う、自動化率30%到達
Agent スタンダード昇格 (¥80,000)
   ↓ 3ヶ月使う、自動化率70%到達
Agent プレミアム昇格 (¥150,000)
   ↓ Agent認定実施、対話モードに興味
★Agent Elite 提案 (¥250,000)
   ↓ Slack連携を試す→刺さる→契約
```

各段階で「次の階段」が見えていることが重要。Eliteを最初から見せておくことで、低価格プランの選択肢としての価値も上がる(=セルフ差別化の効果)。
